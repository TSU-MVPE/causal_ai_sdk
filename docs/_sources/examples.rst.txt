Examples
========

The SDK is used as an async context manager. All service calls go through
``client.kg``, ``client.cd``, or ``client.da``.

Adjacency orientation contract:

- Knowledge Graph ``data`` uses ``cause=row, effect=col``.
- CD result ``adjacency_matrices`` uses ``cause=row, effect=col``.
- ``adjacency_matrices[i][j] != 0`` means ``feature_names[i]`` causes ``feature_names[j]``.
- DA converts to Cation's internal ``effect=row, cause=col`` at ingest.

Knowledge Graph
---------------

Create a session, upload a knowledge graph from file, list and inspect it:

.. code-block:: python

   async with CausalAIClient(api_key="...", base_url="...") as client:
       session = await client.kg.init_session()
       kg = await client.kg.upload_kg_from_file(
           session_uuid=session["uuid"],
           file_path="graph.json",
           title="My Graph",
       )
       kg_list = await client.kg.list_kg(session["uuid"])
       kg_detail = await client.kg.get_kg(session["uuid"], kg["id"])
       # When done: await client.kg.delete_session(session["uuid"])

Causal Discovery (MultiCa)
--------------------------

Upload multiple datasets with ``upload_data_for_multica`` (or use ``get_upload_url``
for a presigned URL and upload yourself). Start matching, then pass the returned
``matching_task_id`` into ``wait_for_matching``, ``run_multica``, and (for cleanup)
``delete_multica_matching``. Use the **run** task id from ``run_multica`` for
``wait_for_task`` and ``get_task_result``:

.. code-block:: python

   async with CausalAIClient(api_key="...", base_url="...") as client:
       session = await client.kg.init_session()
       uploaded = await client.cd.upload_data_for_multica(
           session["uuid"], ["dataset1.csv", "dataset2.csv"]
       )
       matching = await client.cd.start_multica_matching(uploaded)
       matching_task_id = matching["matching_task_id"]
       await client.cd.wait_for_matching(
           session["uuid"], mode="multica",
           matching_task_id=matching_task_id, timeout=300, interval=3
       )
       task = await client.cd.run_multica(
           uploaded, threshold=0.01, roots=["X1"], sinks=["Y"],
           matching_task_id=matching_task_id
       )
       run_task_id = task["task_id"]
       await client.cd.wait_for_task(
           run_task_id, session_uuid=session["uuid"], timeout=300, interval=5
       )
       result = await client.cd.get_task_result(session["uuid"], run_task_id)
       # Cleanup: await client.cd.delete_multica_matching(session["uuid"], matching_task_id)

Causal Discovery (LiNGAM)
-------------------------

Upload a single dataset, run LiNGAM causal discovery, wait for completion, and
fetch the result. Use the **run** task id from ``run_lingam`` for
``wait_for_task`` and ``get_task_result`` (not the upload task id):

.. code-block:: python

   async with CausalAIClient(api_key="...", base_url="...") as client:
       session = await client.kg.init_session()
       uploaded = await client.cd.upload_data_for_lingam(session["uuid"], "data.csv")
       task = await client.cd.run_lingam(uploaded, threshold=0.01)
       run_task_id = task["task_id"]
       await client.cd.wait_for_task(
           run_task_id, session_uuid=session["uuid"], timeout=600, interval=5
       )
       task_result = await client.cd.get_task_result(session["uuid"], run_task_id)
       # task_result contains result_url; fetch JSON from it for feature_names, etc.

Causal Discovery (TraCKR)
-------------------------

Upload a source knowledge graph, upload target data, start TraCKR matching and
pass the returned ``matching_task_id`` into ``wait_for_matching``, optional
``set_trackr_matching``, ``run_trackr``, and (for cleanup) ``delete_trackr_matching``.
Use the **run** task id from ``run_trackr`` for ``wait_for_task`` and
``get_task_result``:

.. code-block:: python

   async with CausalAIClient(api_key="...", base_url="...") as client:
       session = await client.kg.init_session()
       kg = await client.kg.upload_kg_from_file(
           session_uuid=session["uuid"], file_path="source_graph.json"
       )
       uploaded = await client.cd.upload_data_for_trackr(session["uuid"], "target.csv")
       matching_start = await client.cd.start_trackr_matching(uploaded, source_kg_id=kg["id"])
       matching_task_id = matching_start["matching_task_id"]
       await client.cd.wait_for_matching(
           session["uuid"], mode="trackr",
           matching_task_id=matching_task_id, timeout=300, interval=3
       )
       # Optionally: await client.cd.set_trackr_matching(
       #     session["uuid"], {"col_a": "src_a"}, matching_task_id=matching_task_id
       # )
       task = await client.cd.run_trackr(
           uploaded,
           transferred_knowledge={"session_uuid": session["uuid"], "kg_id": kg["id"]},
           threshold=0.01,
           matching_task_id=matching_task_id,
       )
       run_task_id = task["task_id"]
       await client.cd.wait_for_task(
           run_task_id, session_uuid=session["uuid"], timeout=300, interval=5
       )
       result = await client.cd.get_task_result(session["uuid"], run_task_id)
       # Cleanup: await client.cd.delete_trackr_matching(session["uuid"], matching_task_id)

Decision Analysis
-----------------

Run explain or enumerate using a CD result reference. Poll until complete and
fetch the result:

.. code-block:: python

   async with CausalAIClient(api_key="...", base_url="...") as client:
       session = await client.kg.init_session()
       # cd_result_reference = session_uuid + task_id from a completed CD task
       explain_task = await client.da.run_explain(
           session_uuid=session["uuid"],
           cd_result_reference={"session_uuid": "...", "task_id": "..."},
           current_observation={"feature_a": 0.5, "feature_b": 1.0},
           targets=[{"col": "target_col", "sense": ">", "threshold": 0.8}],
       )
       await client.da.wait_for_task(
           session["uuid"], explain_task["task_id"], timeout=300, interval=5
       )
       da_result = await client.da.get_task_result(
           session["uuid"], explain_task["task_id"]
       )
       # Or run_enumerate with same args for enumeration:
       # enum_task = await client.da.run_enumerate(...)

Column matching (MultiCa)
~~~~~~~~~~~~~~~~~~~~~~~~~~

Upload data, start MultiCa column matching, then pass ``matching_task_id`` through
wait, set, run, and delete. Use the run task id for wait/get result.

.. code-block:: python

   from causal_ai_sdk.models import UploadedData

   async with CausalAIClient(api_key="...", base_url="...") as client:
       session = await client.kg.init_session()
       upload_info = await client.cd.get_upload_url(session["uuid"])
       # ... upload your data to S3 using upload_info ...
       uploaded = UploadedData(
           task_id=upload_info.task_id,
           session_uuid=session["uuid"],
           s3_key=upload_info.s3_key,
       )
       matching = await client.cd.start_multica_matching(uploaded)
       matching_task_id = matching["matching_task_id"]
       await client.cd.wait_for_matching(
           session["uuid"], mode="multica",
           matching_task_id=matching_task_id, timeout=300, interval=3
       )
       # Optionally: client.cd.set_multica_matching(
       #     session["uuid"], {"col_a": "src_a"}, matching_task_id=matching_task_id
       # )
       task = await client.cd.run_multica(
           uploaded, threshold=0.01, matching_task_id=matching_task_id
       )
       run_task_id = task["task_id"]
       await client.cd.wait_for_task(
           run_task_id, session_uuid=session["uuid"], timeout=300, interval=5
       )
       result = await client.cd.get_task_result(session["uuid"], run_task_id)
       # Cleanup: await client.cd.delete_multica_matching(session["uuid"], matching_task_id)
